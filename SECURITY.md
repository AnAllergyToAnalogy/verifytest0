# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not open a public GitHub issue.**

Instead, contact us privately using one of the following methods:

- Email: security@yourdomain.com  
- Alternative contact: info@yourdomain.com

If applicable, please include:
- A clear description of the vulnerability
- Steps to reproduce
- Any relevant transaction signatures, program IDs, or logs
- Your assessment of potential impact

We will acknowledge receipt as soon as possible and work to assess and resolve the issue promptly.

---

## Supported Versions

Only the latest deployed version of the on-chain program is considered supported for security updates unless otherwise stated.

---

## Responsible Disclosure

We kindly ask that you:
- Allow us reasonable time to investigate and remediate
- Avoid exploiting the issue beyond proof-of-concept
- Avoid public disclosure until a fix is deployed or agreed upon

We appreciate responsible disclosure and contributions that help keep users safe.

---

## Scope

This policy applies to:
- On-chain programs published from this repository
- Associated off-chain tooling maintained in this repo

Third-party dependencies and external services are subject to their own security policies.
